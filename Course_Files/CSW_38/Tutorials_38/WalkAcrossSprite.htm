Chapter 1: The Principle of Scroll Mapping

In this demonstration, we are using the user's scroll position to control two things simultaneously: the sprite's **horizontal position** across the viewport, and the sprite's **animation frame**. The key concept is mapping the scroll distance (from 0% at the top to 100% at the bottom) to the required values for the animation. This is a common technique in interactive storytelling websites.

To achieve this, the JavaScript continuously calculates the ratio of `current_scroll / total_scroll_height`. If the user has scrolled halfway, the sprite should be halfway across the screen. If the user has scrolled one-quarter of the way, the sprite should display the first quarter of its running cycle.

This content section is intentionally long. Its sole purpose is to provide enough vertical height to allow the page to scroll, giving our JavaScript engine a measurable input to work with.
