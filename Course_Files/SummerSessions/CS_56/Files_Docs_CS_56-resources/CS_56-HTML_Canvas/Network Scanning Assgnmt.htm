🔍 Network Reconnaissance & Tooling

Hands-On Information Gathering Activity

🛡️ The Administrator's Toolkit

Before a network engineer can secure a network, or before a penetration tester can evaluate one, they must first gather intelligence. This phase is known as Network Reconnaissance. In this assignment, you will step into the shoes of a network professional and use a suite of public web tools to analyze domain names, investigate IP addresses, and scan for open ports.

📋 Your Mission Instructions

Navigate to the following external resource. This site provides an expansive toolkit for networking and DNS diagnostics:

Launch DNSChecker Tool Suite 🚀

Select a public target domain (e.g., deltacollege.edu, cisco.com, or an allowed public test server) and complete the following tasks:

  • 1. DNS & WHOIS Lookup: Use the Domain/WHOIS tools to identify the registrar of the domain, the registration date, and the authoritative name servers handling its DNS traffic.
  • 2. IP Location Finder: Find the IP address of your target domain and use the IP tools to determine the physical geographic location of the server.
  • 3. Port Scanner: Run a basic port scan on your target. Document which common ports (like 80 HTTP, 443 HTTPS, 21 FTP, 22 SSH) are left "Open" and which are "Closed" or filtered by a firewall.
  • 4. Ping / Latency Check: Use the network Ping tools to determine the response time (latency) between the testing server and your target.

📝 Deliverables

Create a brief Reconnaissance Report (Word Document or PDF). For each of the four tasks above, include:

  1. A screenshot of your tool results from DNSChecker.
  2. A 2-3 sentence explanation of what that specific information tells you about the target's network architecture or security posture.
⚠️ Ethical Notice: White Hat rules apply. Only conduct reconnaissance on public-facing domains or your own hardware. Do not launch excessive repeated scans against any target.
